# User Flows

## 1. 일반 플레이어 흐름
1. 회원가입/로그인
2. 로비 진입
3. 방 생성 또는 기존 방 참가
4. 준비 상태 변경
5. 호스트가 게임 시작
6. 게임 플레이
7. 결과 확인
8. 방으로 복귀 또는 로비 이동

## 2. follow-owner 봇 연결 흐름
1. 유저가 자신의 봇을 생성/연결
2. 유저가 봇에 사용할 AI 백엔드(OpenClaw 등)를 설정
3. 봇은 "follow-owner" 상태로 대기
4. 유저가 특정 방에 참가
5. 시스템이 owner와 연결된 봇을 확인
6. 봇 참가 조건이 맞으면 같은 방에 봇 참가
7. 게임 도중 봇은 owner 측 팀/상태 규칙을 따라 행동

## 3. 게임 도중 봇 동작 흐름
1. 서버가 현재 게임 상태를 정리
2. 봇 어댑터가 owner 맥락과 게임 상태를 함께 받음
3. AI 백엔드가 행동 결정
4. 서버가 유효성 검증
5. 결과 반영 및 방송

## 4. 실패 흐름
- owner는 방에 들어갔지만 봇 연결이 끊김
- 봇은 연결됐지만 행동 생성 실패
- 행동은 생성됐지만 유효성 검증 실패
- 게임 상태가 바뀌어 해당 행동이 더 이상 유효하지 않음

이 경우 서버는:
- 이유를 로그로 남기고
- 가능한 기본 fallback을 적용하거나
- 턴 스킵/무행동 처리 정책을 따른다.
