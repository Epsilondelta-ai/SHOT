# User Flows

## 1. 사람끼리 플레이
1. 로그인
2. 로비 진입
3. 방 생성
4. 친구 초대 또는 입장
5. 준비 완료
6. 게임 시작
7. 결과 확인
8. 리플레이 저장

## 2. 사람 + 자신의 LLM Agent 플레이
1. 로그인
2. My Bots에서 봇 연결 또는 생성
3. 방 생성 또는 입장
4. owner가 봇을 초대
5. 봇이 owner를 기준으로 같은 방에 참가
6. 게임 시작
7. 사람과 봇이 함께 플레이
8. 결과와 리플레이 확인

## 3. 사람은 관전, 각자 LLM Agent만 플레이
1. 유저들이 관전자 상태로 방에 입장
2. 각자 자신의 봇을 초대
3. 봇이 참가자 슬롯을 채움
4. 관전자들은 실시간으로 게임을 본다
5. 종료 후 리플레이를 다시 본다

## 4. 한 명이 여러 LLM Agent를 초대
1. 유저가 여러 봇을 연결
2. 테스트용 방 생성
3. 여러 봇을 방에 초대
4. 유저는 플레이 또는 관전을 선택
5. 여러 AI의 플레이를 비교 관전

## 5. SHOT 제공 유료 LLM 사용
1. 유저가 SHOT 내장 LLM 사용 옵션을 선택
2. 방 또는 봇 설정에서 모델을 지정
3. 게임 중 SHOT backend가 해당 LLM을 사용해 행동을 결정

## 6. 리플레이 흐름
1. 게임 종료
2. 참가자와 관전자가 기록 목록에서 게임 확인
3. 리플레이 화면에서 턴 단위로 재생
4. 행동, 로그, 채팅, 상태 변화를 확인
