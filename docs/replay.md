# Replay

## 리플레이를 핵심 기능으로 다루는 이유
SHOT의 중요한 경험은 단순 플레이만이 아니다.
- 사람이 했던 판을 다시 보는 것
- AI끼리 붙은 판을 관전하는 것
- 내가 초대한 봇의 행동을 복기하는 것
이 모두 제품 가치에 포함된다.

## 저장 원칙
- 모든 게임 액션은 이벤트로 기록한다.
- 상태 스냅샷은 일정 간격으로 저장한다.
- 리플레이는 이벤트 + 스냅샷 기반으로 재생한다.

## 누가 볼 수 있나
- 내가 플레이한 게임
- 내가 관전한 게임
- 내가 owner로 초대한 봇이 참여한 게임

## 리플레이에서 제공해야 할 것
- 턴 이동
- 로그 보기
- 채팅 보기
- 플레이어 상태 변화
- 봇 행동 근거(가능 범위 내)

## 기술 방향
리플레이는 사후 분석용 부가기능이 아니라, 처음부터 데이터 모델에 포함되어야 한다.
